Convex Hull

Finding the Smallest Convex Polygon Enclosing All Points

Introduction

The Convex Hull is the subset of points that forms the smallest convex polygon which encloses all points in the set. This fundamental concept in computational geometry has numerous applications in computer graphics, pattern recognition, and optimization problems.
To visualize this, imagine that each point is a pole. Then, imagine what happens if you were to wrap a rope around the outside of all the poles, and then pull infinitely hard, such that the connections between any two points that lie on the edge of the rope are lines. The set of points that touch the rope is the convex hull.
Mathematical Definition: A convex hull of a set of points S is the smallest convex polygon (or polytope in higher dimensions) that contains all points in S. The hull is unique and consists of the extreme points of the set.
Key Properties:
‚Ä¢ The convex hull is always a convex polygon
‚Ä¢ It contains all original points either inside or on the boundary
‚Ä¢ Hull vertices are a subset of the original point set
‚Ä¢ The hull has the minimum perimeter among all enclosing convex polygons
‚Ä¢ Any line segment between two points inside the hull lies entirely within the hull
Real-World Applications:
‚Ä¢ Computer Graphics: Collision detection, shape approximation
‚Ä¢ Pattern Recognition: Object boundary detection, clustering
‚Ä¢ Optimization: Linear programming, computational geometry
‚Ä¢ Robotics: Path planning, obstacle avoidance
‚Ä¢ GIS Systems: Geographic boundary computation
‚Ä¢ Image Processing: Shape analysis, feature extraction

Graham Scan Algorithm

Solution

The Graham Scan algorithm works in 3 steps. First, it sorts all of the n points by their counterclockwise angle around a pivot P‚ÇÄ, breaking ties by distance. This algorithm uses the leftmost (and bottommost if there is a tie) point as P‚ÇÄ.
We maintain a stack containing the points such that the following invariant holds: every three consecutive points a, b, c of the stack form a counterclockwise turn. In other words, c lies to the left of the line from a to b. This condition implies that the points of the stack form the vertices of a convex polygon.
To start the creation of the convex hull, we choose 2 points. The pivot (first point), and the second point based on our initial sorting. After that, we attempt to add each point in cand to the stack.

Denote our stack as hull, the top element of hull as hull[i] and cand[j] as the j-th sorted candidate point. Before adding cand[j] to the stack, we check whether hull[i-1] ‚Üí hull[i] ‚Üí cand[j] forms a counterclockwise turn.

  1. If so, then we add cand[j] to the stack and the invariant holds. Continue with cand[j+1].
  2. Otherwise, hull[i] lies within the convex hull of the other points in the stack along with cand[j], so we pop hull[i] from the stack and continue with cand[j].

Worked Example

Find the convex hull of the points (1,2), (2,3), (5,3), (3,2), (2,0), (6,2).
Animation Reference: This example can be visualized with various online tools. Interactive animations help understand the step-by-step process of Graham Scan.

Using the steps from our Graham Scan:

1. Sorting the points by counterclockwise angle: Our leftmost point is (1,2), so we then sort all the points by their counterclockwise angle. If the angles are the same, we can tiebreak by distance. Our resulting sorted points are (2,0), (6,2), (3,2), (5,3), (2,3) with initial pivot (1,2). Note that (6,2), (3,2) are collinear but since (6,2) is further away it breaks the tie, thus placing it before (3,2).
2. Maintaining the stack: Our initial stack elements are (1,2), (2,0). Then, starting from cand[2], we process each point.
  1. First, we check if the turn between (1,2), (2,0), and (3,2) is counterclockwise, which it is. Hence, the point cand[2]=(3,2) is inserted on our stack. Our stack now corresponds to the points: (1,2), (2,0), (3,2).
  2. Then, we check the turn (2,0), (3,2), (6,2), which is clockwise. Hence, we pop the stack until the counterclockwise property can be satisfied. In this case, only (3,2) is popped. Afterwards, we push (6,2) to our stack because it satisfies the counterclockwise property. Our stack now corresponds to the points: (1,2), (2,0), (6,2).
  3. Then, we check whether the turn between (2,0), (6,2), (5,3) is counterclockwise, which it is, so we push (5,3) to our stack. Our stack now corresponds to the points: (1,2), (2,0), (6,2), (5,3).
  4. Then, we check whether the turn between (6,2), (5,3), (2,3) is counterclockwise, which it is, so we push (2,3) to our stack. Since we have processed all points, we have completed the cycle, and our convex hull is complete.
Our stack now is then (1,2), (2,0), (6,2), (5,3), (2,3), which are the vertices of the convex hull.
Time Complexity: $\mathcal{O}(n \log n)$ for sorting + $\mathcal{O}(n)$ for scanning = $\mathcal{O}(n \log n)$
Space Complexity: $\mathcal{O}(n)$ for the stack
C++ - Graham Scan Implementation 
#include <bits/stdc++.h>

using namespace std;

#define FOR(i, a, b) for (int i = (a); i < (b); i++)
#define FORE(i, a, b) for (int i = (a); i <= (b); i++)
#define F0R(i, a) for (int i = 0; i < (a); i++)
#define trav(a, x) for (auto &a : x)

#define f first
#define s second
#define bk back()
#define pb push_back

#define sz(x) int((x).size())
#define bg(x) begin(x)
#define all(x) bg(x), end(x)

int N, Q;

const int MX = 2e5 + 5;

using T = long long;
const T EPS = 1e-9;
using P = pair<T, T>;
using vP = vector<P>;
using Line = pair<P, P>;

T sq(T a) { return a * a; }
T norm(const P &p) { return sq(p.f) + sq(p.s); }

P operator-(const P &l, const P &r) { return P(l.f - r.f, l.s - r.s); }

T cross(const P &a, const P &b) { return a.f * b.s - a.s * b.f; }
T cross(const P &p, const P &a, const P &b) {
    return cross(a - p, b - p);
}

using vi = vector<int>;
using vP = vector<P>;

vi hullInd(const vP &v) {
    int ind = int(min_element(all(v)) - begin(v));
    vi cand, hull{ind};
    F0R(i, sz(v)) if (v[i] != v[ind]) cand.pb(i);

    sort(all(cand), [&](int a, int b) {
        P x = v[a] - v[ind], y = v[b] - v[ind];
        T t = cross(x, y);
        return t != 0 ? t > 0 : norm(x) < norm(y);
    });

    trav(c, cand) {
        while (sz(hull) > 1 && cross(v[end(hull)[-2]], v[hull.bk], v[c]) <= 0) {
            hull.pop_back();
        }
        hull.pb(c);
    }

    return hull;
}

int main() {
    ios_base::sync_with_stdio(0);
    cin.tie(0);

    for (;;) {
        int n;
        cin >> n;
        if (!n) break;
        vP pts(n);

        F0R(i, n) {
            long long a, b;
            cin >> a >> b;
            pts[i] = P(a, b);
        }

        vi ret = hullInd(pts);

        cout << sz(ret) << '\n';
        trav(a, ret) {
            cout << pts[a].first << " " << pts[a].second << '\n';
        }
    }
}

Monotone Chain Algorithm

Solution

With the Monotone Chain algorithm, we start by sorting the given n points in ascending order with respect to their x coordinates. If two points have the same x coordinate, then we will look at the y coordinate.
Next, we will calculate the convex hull in two parts - the upper and the lower hull. Firstly, we observe that the starting and ending points of both upper and lower hulls are the same. They are the points with the lowest and highest x value respectively, P‚ÇÄ and P‚Çô‚Çã‚ÇÅ.

We start by adding P‚ÇÄ and P‚ÇÅ to the hull. (Note that P‚ÇÅ doesn't necessarily have to be on the convex hull at the end). Then, starting from P‚ÇÇ, we iterate through the sorted points and add them to the hull. Let's denote the current point being added as P‚Çñ and the last point still on the hull as P·µ¢.

When adding new points, we want to make sure that there is no right turn among all segments of the hull, just like in the Graham Scan algorithm discussed above. To achieve this, the segment P·µ¢‚Çã‚ÇÅP·µ¢ should always be on the right side of the segment P·µ¢‚Çã‚ÇÅP‚Çñ. This can be calculated by using a cross-product:
  1. If (P·µ¢ - P·µ¢‚Çã‚ÇÅ) √ó (P‚Çñ - P·µ¢‚Çã‚ÇÅ) < 0, the point P·µ¢ lies on the left side of the segment P·µ¢‚Çã‚ÇÅP‚Çñ. In this case, we have to remove point P·µ¢ from the hull and repeat this check.
  2. If (P·µ¢ - P·µ¢‚Çã‚ÇÅ) √ó (P‚Çñ - P·µ¢‚Çã‚ÇÅ) = 0, the point P·µ¢ lies on the segment P·µ¢‚Çã‚ÇÅP‚Çñ. Whether to include multiple collinear points depends on the question, but for the given question above, we will remove the point P·µ¢ as well and repeat the check.
  3. Otherwise, the point P·µ¢ lies on the right side of the segment P·µ¢‚Çã‚ÇÅP‚Çñ. In this case, we can add P‚Çñ to the hull and process the next point P‚Çñ‚Çä‚ÇÅ from the given point list.
After all the points have been processed, we have found the lower hull and will begin to find the upper hull in the same manner. This time, we add point P‚Çô‚Çã‚ÇÇ to the hull and iterate from the end of the points, P‚Çô‚Çã‚ÇÉ, to the starting point P‚ÇÄ. (The point P‚Çô‚Çã‚ÇÇ also doesn't necessarily have to be the convex hull and could be removed if it causes a right turn).
At the end, we have got all points of the convex hull in the counterclockwise order. To do this in the clockwise order, one only has to change the condition for (P·µ¢ - P·µ¢‚Çã‚ÇÅ) √ó (P‚Çñ - P·µ¢‚Çã‚ÇÅ) from more than 0 to less than 0. In this case, the upper hull will be found first and then the lower hull.
Time Complexity: $\mathcal{O}(n \log n)$ to sort the points + $\mathcal{O}(n)$ to calculate the hull = $\mathcal{O}(n \log n)$
Space Complexity: $\mathcal{O}(n)$

Worked Example

Let's consider the same example from the Graham Scan algorithm. Find the convex hull of the points (1,2), (2,3), (5,3), (3,2), (2,0), (6,2).

Following the steps of the Monotone Chain algorithm:

1. Sorting the given points primarily by their x coordinates and secondarily by their y coordinates. The sorted list would be (1,2), (2,0), (2,3), (3,2), (5,3), (6,2)
2. Find the lower hull:
  1. Firstly, we add the first two points into the hull. The list representing the hull would now contain these two elements [(1,2), (2,0)]. Let's start iterate through the rest of the points. Our P‚Çñ now is (2,3). The cross product, ((2,0) - (1,2)) √ó ((2,3) - (1,2)) = (1,-2) √ó (1,1) = (1 ¬∑ 1) - (-2 ¬∑ 1) = 3, is more than 0, which tells us that there is no right turn, and we can move to our next point, (3,2). The list now contains [(1,2), (2,0), (2,3)].
  2. The cross product ((2,3) - (2,0)) √ó ((3,2) - (2,0)) = (0,3) √ó (1,2) = (0 ¬∑ 2) - (3 ¬∑ 1) = -3 is less than zero, meaning there is a right turn from (2,0)(2,3) to (2,3)(3,2). Therefore, we remove point (2,3) from our hull. The hull now contains [(1,2), (2,0)]. We then test the cross product ((2,0) - (1,2)) √ó ((3,2) - (1,2)) = (1,-2) √ó (2,0) = (1 ¬∑ 0) - (-2 ¬∑ 2) = 4, which is positive, so we add point (3,2) to our hull. The hull now contains [(1,2), (2,0), (3,2)].
  3. We keep testing the remaining points. We want to know if there is a right turn from (2,0)(3,2) to (3,2)(5,3), and the cross product ((3,2) - (2,0)) √ó ((5,3) - (2,0)) = (1,2) √ó (3,3) = (1 ¬∑ 3) - (2 ¬∑ 3) = -3 is negative, so there is a right turn. We remove point (3,2). Next, we test the two segments, (1,2)(2,0) and (2,0)(5,3). The cross product is ((2,0) - (1,2)) √ó ((5,3) - (1,2)) = (1,-2) √ó (4,1) = (1 ¬∑ 1) - (-2 ¬∑ 4) = 9, which is positive, so there is a left turn. We add point (5,3) and move to the next point. Our list looks now like this: [(1,2), (2,0), (5,3)].
  4. Then, we test the segments (2,0)(5,3) and (5,3)(6,2). The cross product ((5,3) - (2,0)) √ó ((6,2) - (2,0)) = (3,3) √ó (4,2) = (3 ¬∑ 2) - (3 ¬∑ 4) = -6 is negative, so there is a right turn. We remove (5,3) from the list and now test the segments (1,2)(2,0) and (2,0)(6,2). The cross product is ((2,0) - (1,2)) √ó ((6,2) - (1,2)) = (1,-2) √ó (5,0) = (1 ¬∑ 0) - (-2 ¬∑ 5) = 10, which is positive, so there is a left turn. We add point (6,2) to our hull. The hull now contains [(1,2), (2,0), (6,2)]. As point (6,2) is the last element in the list, we have found the lower hull of the given set of points.
3. Find the upper hull:
  1. We first add the point (5,3) to the end of the list and process the point (3,2). The cross product for ((6,2) - (5,3)) √ó ((3,2) - (5,3)) = (1,-1) √ó (-2,-1) = (1 ¬∑ (-1)) - (-1 ¬∑ (-2)) = -3 is negative, meaning there is a right turn, so we remove point (5,3) from the list. We then test ((6,2) - (6,2)) √ó ((3,2) - (6,2)), but since there aren't enough points left in the upper hull, we add point (3,2) to the list. The hull contains [(1,2), (2,0), (6,2), (3,2)].
  2. Then we process the next point (2,3). The cross product for segments (6,2)(3,2) and (3,2)(2,3) is ((3,2) - (6,2)) √ó ((2,3) - (6,2)) = (-3,0) √ó (-4,1) = (-3 ¬∑ 1) - (0 ¬∑ (-4)) = -3, which is negative, so there is a right turn. We remove point (3,2). Testing again with (6,2) and (2,3), we need a point before (6,2) which is (2,0). The cross product ((6,2) - (2,0)) √ó ((2,3) - (2,0)) = (4,2) √ó (0,3) = (4 ¬∑ 3) - (2 ¬∑ 0) = 12 is positive, so there is a left turn. We can add point (2,3) to our hull. The list contains [(1,2), (2,0), (6,2), (2,3)].
  3. The next point is (2,0). The segments (6,2)(2,3) and (2,3)(2,0) have cross product ((2,3) - (6,2)) √ó ((2,0) - (6,2)) = (-4,1) √ó (-4,-2) = (-4 ¬∑ (-2)) - (1 ¬∑ (-4)) = 12, which is positive, so there is a left turn. We can add point (2,0) to our hull. The list contains [(1,2), (2,0), (6,2), (2,3), (2,0)].
  4. The next point is (1,2). There is a right turn from (2,3)(2,0) to (2,0)(1,2), so we remove point (2,0) from the list. We test again for segments (6,2)(2,3) and (2,3)(1,2). The cross product is ((2,3) - (6,2)) √ó ((1,2) - (6,2)) = (-4,1) √ó (-5,0) = (-4 ¬∑ 0) - (1 ¬∑ (-5)) = 5, which is positive, so there is a left turn. We add point (1,2) to the list. We have reached the beginning of our list so we can terminate here. As (1,2) was added while calculating the lower hull, we then remove the last element, i.e. the added point (1,2), from the list.
Now, we have the upper hull calculated as well. The final result is [(1,2), (2,0), (6,2), (2,3)]. These are the vertices of our convex hull in counterclockwise order.
C++ - Monotone Chain Implementation 
#include <bits/stdc++.h>
using namespace std;

using pii = pair<int, int>;

vector<pii> points;
vector<pii> hull;

// cross product, the signed area of these three points
int area(pii O, pii P, pii Q) {
    return (P.first - O.first) * (Q.second - O.second) -
           (P.second - O.second) * (Q.first - O.first);
}

void monotone_chain() {
    sort(points.begin(), points.end());
    points.erase(unique(points.begin(), points.end()), points.end());
    int n = points.size();

    if (n < 3) {
        hull = points;
        return;
    }

    for (int i = 0; i < n; i++) {
        while (hull.size() > 1 &&
               area(hull[hull.size() - 2], hull.back(), points[i]) <= 0)
            hull.pop_back();
        hull.push_back(points[i]);
    }

    auto lower_hull_length = hull.size();
    for (int i = n - 2; i >= 0; i--) {
        while (hull.size() > lower_hull_length &&
               area(hull[hull.size() - 2], hull.back(), points[i]) <= 0)
            hull.pop_back();
        hull.push_back(points[i]);
    }

    hull.pop_back();
}

int main() {
    cin.tie(0)->sync_with_stdio(false);

    int n;
    cin >> n;
    while (n != 0) {
        points.assign(n, {});
        hull = {};
        for (auto &p : points) cin >> p.first >> p.second;
        monotone_chain();

        cout << hull.size() << "\n";
        for (auto &p : hull) cout << p.first << " " << p.second << "\n";

        cin >> n;
    }

    return 0;
}

Advanced Concepts and Comparisons

Algorithm Comparison

Graham Scan vs Monotone Chain:
‚Ä¢ Graham Scan: Requires polar angle sorting, handles collinear points differently
‚Ä¢ Monotone Chain: Uses Cartesian coordinate sorting, more stable numerically
‚Ä¢ Both: O(n log n) time complexity, O(n) space complexity
‚Ä¢ Practical: Monotone Chain is often preferred for its simplicity and robustness
Implementation Considerations:
‚Ä¢ Use integer coordinates when possible to avoid floating-point precision issues
‚Ä¢ Handle degenerate cases: collinear points, duplicate points, single points
‚Ä¢ Consider whether to include collinear points based on problem requirements
‚Ä¢ Ensure correct handling of tie-breaking in sorting
‚Ä¢ Test with edge cases: all points collinear, points forming a line, etc.

Optimization Techniques

Cross Product Optimization: Use integer arithmetic for cross products when dealing with integer coordinates. The cross product (a √ó b) = a‚Çìb·µß - a·µßb‚Çì can be computed efficiently without floating-point operations.
Sorting Optimization: For large datasets, consider using radix sort or other linear-time sorting algorithms when coordinates have limited range. For typical competitive programming problems, standard O(n log n) sorting is sufficient.

Extensions and Variations

3D Convex Hull: The concepts extend to higher dimensions, but algorithms become more complex. QuickHull and other algorithms are used for 3D and higher-dimensional convex hulls.
Dynamic Convex Hull: For applications requiring insertion and deletion of points, specialized data structures maintain the convex hull dynamically with better amortized complexity.
Approximate Convex Hull: For very large datasets, approximate algorithms can provide near-optimal solutions with better performance characteristics.

Summary and Key Takeaways

Convex hull algorithms are fundamental tools in computational geometry. Both Graham Scan and Monotone Chain provide efficient O(n log n) solutions with different strengths and implementation characteristics.
